# 00 — 產品定位與範圍

## 定位

- **一句話**：研究所考生的「備考作戰中心」——把散落在補習班官網 / PTT / 各校招生處的資訊，整合成結構化、可互動、有 AI 陪練的單一入口。
- **價值主張**：從「該考哪間、怎麼準備」到「考古題練到會」，一站搞定。
- **差異化護城河**：AI 解題（標準解答 + 知識點延伸），補習班與通用 ChatGPT 都給不了的結構化加值。
- **情境**：台灣研究所考試。起步聚焦 **資管 / 資工**（考科高度重疊，共用題庫效益大）。

## 競品定位

| 類型       | 代表            | 缺口（我們的機會）                      |
| ---------- | --------------- | --------------------------------------- |
| 補習班入口 | 大碩、TKB       | 內容是導購漏斗、UI 老舊、無互動解題     |
| 線上題庫   | 阿摩            | 研究所專業科目薄弱、解析品質參差、無 AI |
| 落點/榜單  | 1111、104       | 研究所顆粒度粗                          |
| 經驗分享   | PTT、Dcard、IOH | 非結構化、難搜尋、易過時                |

定位：不做「又一個題庫網」，做整合入口 + AI 陪練。

## 內容政策（合規，已定案）

- 歷屆考題**僅收錄官方已公開者**，每題保留 `source_url` + `license_status`：
  - `national_exam`：政府公文性質，可完整收錄。
  - `school_official`：學校官方公開，依各校條款，附來源連結。
  - `unknown`：不收。
- **純非營利**：不收費、不放廣告（降低合理使用爭議）。
- 標示「AI 生成解析，僅供參考」，提供錯誤回報與 DMCA 取下機制。
- 可主動寫信向系所索取授權（白名單可完整收錄）。

> 重點：附來源連結不等於取得重製授權；非營利只是合理使用四要素之一。完整照搬全校系大量考題即使免費仍有風險，故採「官方公開 + 連結」策略，價值放在 AI 解析（我們的原創）。

## 功能模組與優先級

### MVP

1. 考古題瀏覽庫（類組/學校/科目/年份 結構化檢索）。
2. AI 解題（單題：標準解答 + 知識點）。
3. 報名情報頁 + CTA（簡章、時程、近年報名錄取比，含圖表）。
4. 報名行事曆 + deadline 提醒訂閱。

### 第二階段

1. 線上答題 / 模考系統（計時、自動批改、做題記錄）。
2. 弱點分析儀表板（知識點雷達圖、趨勢）。
3. 知識點延伸對話（串知識圖譜）。

### 第三階段

1. 眾包解題 + 審核評分。
2. 落點 / 錄取機率估算。
3. 社群（校系討論、讀書計畫、上榜心得）。

## 商業/營運模式

純非營利。AI 成本以「離線預生成 + 快取」壓到近零邊際；線上追問走 BYOK 或嚴格 rate limit；
伺服器/API 成本可用 GitHub Sponsors / Ko-fi 等捐贈支應（非廣告，不影響非營利定位）。
