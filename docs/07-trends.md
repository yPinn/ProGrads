# 07 — 考科趨勢（規劃）

對應 [00-product.md](00-product.md) 第二階段「弱點分析儀表板（知識點雷達圖、趨勢）」的具體化。
本文件鎖定**考科趨勢**的資料相依、分階段落地、指標定義與風險 guardrail，作為實作單一真相。

## 定位

- **一句話**：把歷屆考題的**考點／題型**沿年份與學校樞紐化，給考生「近 N 年高頻考點、哪些在升溫」的**數據化**備考方向。
- **差異化**：補習班給口耳考古方向、阿摩無分析；我們給矩陣。這是護城河的延伸（衍生聚合、我方原創 IP、合規安全）。
- **兩種切法**（同一引擎）：
  - **a 單校單考科**：某校某卷歷年考點×年 + 題型×年。→ 針對性最強。
  - **b 多校單考科**：同考科跨校比較（考點×校、共通高頻考點）。→ 讀書優先序。

## 資料現況決定可行性

趨勢三軸成熟度不一（以現有題檔實測）：

| 軸           | 來源                               | 現況                                                                                                                                                                 |
| ------------ | ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 題型組成趨勢 | `question_type`                    | 已入庫，可算                                                                                                                                                         |
| 細分科目分佈 | `question_subject`（subject slug） | 已入庫，可算                                                                                                                                                         |
| **考點趨勢** | frontmatter `knowledge_points`     | 檔案裡有（覆蓋 100%），但**尚未入庫**（見 [03-content-pipeline.md](03-content-pipeline.md) frontmatter、[content.ts](../packages/shared/src/content.ts) 標 phase 2） |

考點是最有價值的一軸，但有兩個前置：**入庫 migration** 與 **受控詞彙表**（見下 §風險 1、5）。

## 分階段落地

| 階段             | 交付                                                                                                                            | 前置            | 動 schema                     |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------- | --------------- | ----------------------------- |
| **P0（已完成）** | 離線 `report-trends`，當補題羅盤                                                                                                | 無（KP 已在檔） | 否                            |
| **P0.5**         | knowledge-point 受控詞彙表（兩層：主題→子題）；補題時引用既有 slug，隨補題漸進做                                                | slug 登記機制   | 否（純檔案層）                |
| **P1**           | `KnowledgePoint` 目錄表 + `question_knowledge_point` join；sync 建立；validator gate KP slug；一次性 backfill（變體→canonical） | P0.5 詞彙穩定   | 是（比照 `question_subject`） |
| **P2**           | API `a` `GET /v1/trends/:subject?school=`、`b` `GET /v1/trends/:subject`；Web 考科趨勢頁（考點熱力圖 + 題型堆疊 + 跨校比較）    | P1 + 縱深達標   | 否（GROUP BY）                |

**觸發閘門**：P1/P2 不提早動——只有當「**≥N 卷達 ≥3 年**」才有東西可展示、詞彙才在真資料上穩下來。在那之前只用 P0 工具導引補題**縱深**（同校同科多年份）。
**P0.5 與趨勢脫鉤也有價值**（同時餵「依考點練習」與弱點雷達圖），故先做不浪費。

## 離線工具（P0，唯讀、無 DB）

見 [03-content-pipeline.md](03-content-pipeline.md) §參照 vs 內容實體：

- `report-questions <dir> [--gaps] [--md]`：覆蓋率（校/年/卷、題型、解題完成度、缺題學校）。
- `report-trends <dir> [--paper=<slug>] [--school=<slug>] [--top=N] [--by-points] [--md]`：考點×年、題型×年、考點×校樞紐；`--paper` 缺省時列各卷年份縱深，標 ≥3 年（趨勢可讀）。

實測驗證：NTU `english-a` 4 年跑出可讀真趨勢（語境填空 15→26→35→28 ↑、克漏字 15→16→15→5 ↓）；`dsa` 僅 2 年，箭頭僅供參考。

## 指標定義（P2 產品必守）

- **預設用「佔卷比（% of paper）」而非原始次數**：原始次數會把「考點變多」與「該卷題數變多」混淆。
- **一題多考點**：欄位總和會超過題數；須明確定義為「每題出現」計 1（去重）或「每標籤」計 1，UI 標示所採口徑。
- **配分加權為選項**（`--by-points`）：一題 30 分申論 vs 五題 2 分選擇，次數會低估權重；但 `points` 常缺，加權為輔非主。
- **最小年份閘門**：< 3 年不顯示趨勢箭頭，只顯示分佈並標「資料不足」。

## SWOT

**S**：唯一數據化差異點（護城河延伸）；資料已存在、P0 已證；衍生聚合合規安全、我方 IP；複用既有 join/slug/report 模式；一魚兩吃（練習 + 弱點雷達）。
**W**：現況多為 1–2 年/卷，趨勢統計上無意義；KP 詞彙碎片化 + AI 標註漂移，趨勢被標註方式污染；跨校 paper slug 相同 ≠ 考科範圍相同；P1 需 migration + backfill + 詞彙治理；指標定義含糊。
**O**：「近 N 年高頻考點」高使用者價值 + SEO 金句；同資料餵弱點雷達圖；策展飛輪（工具指出薄處→補題→趨勢更準）。
**T**：資料薄時上線會誤導考生 → 非營利學習站信任損傷；大型受控詞彙維護負擔；過早產品化造成廣度漂移（抽走題庫縱深火力）；考綱調整/收錄視窗造成假趨勢難澄清。

## 潛在問題與 guardrail（實作前須內建）

1. **分母 + 多標重複計數**：預設佔卷比、明確口徑（見 §指標定義）。
2. **AI 標註漂移 + 版本綁定**：KP 來自我方模型對解答的標註，非官方考卷；換模型重生解答會位移歷史 KP。KP 標註**綁 model 版本**，重標**整批**做、不零星補。
3. **跨校可比性假設**：`b` 目前靠 paper slug 字串對齊（nchu dsa vs ntu dsa），但範圍未必一致（某校可能併入 OS）。需一層 **canonical 考科分類映射**，不能只比 slug。
4. **收錄視窗 = 偽歷史**：只有自己策展的年份；跳過的年份會讓「趨勢」淪為收錄窗口。UI 誠實標註**資料量/信心**，設最小年份閘門；描述性、非預測性。
5. **詞彙治理規模**：光 `dsa` 就 200+ 考點，全站恐 500+；flat slug 會失控 → 採**兩層分類法（主題→子題）**。可能是最大隱藏成本。
6. **Backfill 順序性**：必須**先**把現有自由文字 KP 映射到 canonical **才能**入庫，否則 DB 繼承碎片。P1 前一次性硬工。
7. **責任框架**：呈現趨勢隱含「這科要讀 X」；薄資料出錯 = 信任傷害。文案描述性、常駐信心標示。

## 現階段定案

- **P0 工具已足夠**，純當補題羅盤，不開產品面、不動 schema——避免廣度漂移。
- **優先投在題庫縱深**（同校同科多年份），縱深同時餵飽題庫（MVP 護城河）與趨勢兩個目標。
- **P0.5 受控詞彙隨補題漸進做**（獨立於趨勢也有價值）。
- **P1/P2 以「≥N 卷達 ≥3 年」為觸發閘門**，啟動時把上列 7 點做成硬性 guardrail。
