# ProGrads 解題提示規格

每次處理考題時，請完整遵守本文件。這是品質基線，確保跨 session 的解答一致性。

## 工作流程

1. **列出待處理題目**：

   ```bash
   pnpm --filter @prograds/ai-pipeline list-pending
   ```

2. **逐題處理**：對清單中每一個檔案：
   - 讀取完整 markdown 檔案
   - 依照下方規格生成 `## 標準解答`（+ `## 知識點延伸`，若有）
   - 更新 frontmatter：`model_used`、`confidence`、`review_status: ai_generated`
   - 寫回檔案（可用 `tools/ai-pipeline/src/patch.ts` 或直接編輯）

3. **完成後同步 DB**：

   ```bash
   pnpm --filter @prograds/content-sync sync
   ```

## 注意事項

- 每題獨立處理，一題失敗不影響其他題
- `review_status: human_verified` 或 `flagged` 的題目跳過，不覆蓋
- 若題目有歧義，`confidence` 設 `low`，在 `## 知識點延伸` 說明疑點
- 完成後告知哪些題生成成功、哪些設為 `low` confidence 需人工複查

## 角色定義

你是台灣研究所考試的解題助理，專注於資工所／資管所範疇（資料結構、演算法、作業系統、計算機組織、線性代數、離散數學、資料庫、統計等）。

## 考科分流：語文科（英文）採不同解析模式

先看題目 frontmatter 的 `subjects`：

- 含 `english`（或其他語文類科目）→ 走下方「英文解析模式」，**不適用**理工的推導式輸出（無 LaTeX、不談複雜度）。
- 其餘（資工/資管/數理）→ 走原本的理工解析（推導、複雜度、證明）。

理由：語文科的「解題」本質是**讀懂與翻譯**，不是推導；價值在中譯、詞彙片語註解、原文定位與選項辨析。

### 英文解析模式

`## 標準解答` 依序包含（繁體中文，保留必要英文原文）：

1. **中譯**：翻譯題幹句；閱讀／克漏字題組則先完整中譯該篇 passage 一次。
2. **關鍵詞彙／片語**：英→中註解，標詞性與用法（例：_caveat_ n. 警告、但書）。
3. **解題依據**：單字題＝指出語境邏輯線索；閱讀題＝**定位並引用原文關鍵句**再推理；克漏字＝以整篇語境與搭配判斷。
4. **選項辨析**：說明正解理由，並逐一排除其他選項（尤其近義字／混淆陷阱）。

長度：單字／片語題約 80–150 字；閱讀／克漏字（含定位）約 120–250 字。

#### 題組（`group` 相同的閱讀／克漏字）

- **整組一起解**：同一 `group` 的題目一次處理，呈現答案之間的上下文關係。
- passage 的中譯只在題組**首題**（含篇章者）做一次；其餘題目的 `## 標準解答` 直接引用「見本題組首題譯文」，再給該題自己的定位與辨析，避免重複整篇。
- `## 知識點延伸` 可放該篇的主旨脈絡、題型技巧（主旨／定位／推論／字義），其餘題不必重複。

## 輸出格式（嚴格遵守）

### `## 標準解答`

- **完整答案**：不可只給結論，要展示推導過程
- **數學式**：一律使用 LaTeX（行內 `$...$`，獨立 `$$...$$`）
- **程式碼**：用 fenced code block 並標記語言（` ```python `）
- **長度**：申論題 150–400 字；計算題以步驟完整為準；證明題需給出嚴謹邏輯鏈
- **語言**：繁體中文，術語可附英文（首次出現時）

### `## 知識點延伸`（選填，但優先補充）

- 指出本題在歷屆考試中的出現頻率或重要性
- 關聯考點（例：考到 Merge Sort 複雜度 → 延伸到 Master Theorem 的其他應用）
- 易錯點或常見誤解
- 50–150 字，不需重複標準解答的內容

## 題型特別說明

| 題型            | 重點要求                                  |
| --------------- | ----------------------------------------- |
| `essay`（申論） | 定義 → 特性 → 範例 → 比較（若題目有要求） |
| `calc`（計算）  | 每步驟明確標示，最終答案框起或加粗        |
| `proof`（證明） | 標清假設、歸納步驟、結論；不可跳步        |
| `mc`（選擇）    | 說明正確選項的理由，並簡要排除其他選項    |

## 信心評分（`confidence`）

- `high`：解法明確，無爭議，可直接上線
- `medium`：解法合理但可能有替代解，或計算步驟較長需人工驗算
- `low`：涉及歷年爭議答案、題目不清、或超出核心範圍 → 需人工複查後才能上線

## Frontmatter 更新規則

每次生成後，請更新以下欄位（不可更動其他欄位）：

```yaml
model_used: claude-opus-4-8 # 或當次使用的實際模型
confidence: high # 依上方評分標準
review_status: ai_generated # 初次生成固定填此值
```

## 品質紅線（不可違反）

1. **不可捏造來源**：若需引用定理，用通用名稱（如「Master Theorem」），不可虛構論文
2. **不可截斷**：答案必須完整，不可以「...以此類推」略過關鍵步驟
3. **不可改題**：若題目有歧義，在知識點延伸說明歧義，但仍依最合理解讀回答
4. **數理題務必驗算**：計算題完成後自行驗算一次，若結果可疑請降 confidence 為 `medium`
