# ProGrads 解題複查規格

複查角色（Codex 或人工）的檢查清單。品質基線與 [PROMPT-generate.md](./PROMPT-generate.md) 共用——本檔**不重抄**生成規格，只列
複查時要獨立核對的項目。複查是**查錯**，不是重寫；發現問題就標記/修正，不需要重新生成整份解答。

## 前置

複查對象：已由 Claude Code session 依 PROMPT-generate.md 生成、`review_status: ai_generated` 的題目。

## 檢查項目

### 1. 正確性（核心）

獨立驗算，不照單全收生成者展示的推導：

- `calc`（計算）：重算關鍵步驟，確認最終答案與單位正確。
- `proof`（證明）：檢查邏輯鏈是否有跳步，假設是否成立。
- `mc`（選擇）：覆核正解，確認排除其他選項的理由站得住。
- `essay`（申論）：定義／特性／範例／比較（若題目要求）是否到位、無錯誤陳述。

### 2. Grounding / 不捏造

- 引用定理用通用名稱（如「Master Theorem」），沒有虛構論文或來源。
- 沒有「…以此類推」之類的截斷，關鍵步驟完整。
- 沒有改動題目原意。

### 3. 格式合規

- LaTeX 正確可渲染（行內 `$...$`、獨立 `$$...$$`），無語法錯誤。
- 程式碼區塊標語言（如 ` ```python `）。
- Markdown 結構完整，`## 標準解答` / `## 知識點延伸` 標題與縮排未壞。

### 4. 科目分流合規

先看 frontmatter `subjects`：

- 含語文科（如 `english`）→ 應走英文解析模式（中譯／關鍵詞彙／解題依據／選項辨析），**不應**出現理工推導式輸出。
- 理工/數理科 → 應走推導模式（展示過程、複雜度、證明），不是只給結論。
- 套錯模式（例如語文題被套用理工格式）視為格式錯誤，退回修正。

### 5. Confidence 校準

生成者標的 `confidence` 是否站得住，必要時**降級**（複查只能降不能升，升級需真人簽核見下）：

- 有合理替代解讀、歷年爭議答案、或依賴未明講的假設 → 不得為 `high`，至少 `medium`。
- 題目本身不清或超出核心範圍 → `low`。
- 生成者若高估（例如爭議題卻標 `high`），複查需降級並在 `## 知識點延伸` 補說明爭點。

### 6. 歧義與假設處理

- 若答案依賴未明講的模型假設，`## 標準解答` 開頭是否已列出採用假設。
- 字面嚴格解讀 vs 常見教材語境衝突時，是否保留 caveat（不可寫成絕對無爭議）。

### 7. Frontmatter

- `model_used`、`confidence`、`review_status` 三欄存在且格式正確。
- **複查通過** → `review_status: ai_reviewed`。
- **複查發現問題並已修正** → 視情節維持 `ai_generated`（待再次複查）或設 `flagged`。
- **`review_status` 紅線**：複查（含 Codex）本身**不得**把 `review_status` 升為 `human_verified`——該值只能由
  真人明確簽核動作設定，不是複查通過的自動結果。`ai_reviewed` 代表「AI 複查過」而非「人看過」（見
  [04-ai-pipeline.md](../../docs/04-ai-pipeline.md) 品質與信任）。

## 輸出

複查完一批後，回報：

- 通過的題目清單（無需變動）。
- 需要降 confidence 的題目與理由。
- 需要修正內容或標 `flagged` 的題目與理由。
